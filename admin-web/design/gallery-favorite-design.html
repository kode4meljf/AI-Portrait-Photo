一、收藏按钮位置（以全店卡片逻辑为准）

优先级位置说明
P0 大图右上角 hero-chrome 与「1/3」「全部下载」同一行，顺序：张数 → 收藏 → 下载
未收藏：☆ 收藏 白底 pill
已收藏：★ 已收藏 浅金底 + 描边
P1 全屏预览底栏 关闭 × 与下载 ↓ 之间增加 ★，行为与主入口同步
P1 底部风格缩略图角标 仅展示态：已收藏的风格 thumb 右上角显示 ★（不可点击，避免误触）

二、交互规则

  • 收藏粒度：单张写真（photos.isFavorite），随 swiper 当前张切换状态
  • 仅 generateStatus === completed 且有成片时显示收藏按钮；生成中 / 失败隐藏
  • 点击切换收藏 → 写库 photos.doc(photoId).update({ isFavorite }) → toast「已收藏 / 已取消收藏」
  • 云相册「按收藏」Tab 已按 isFavorite: true 筛选批次，收藏后该批次出现在此 Tab
  • 从云相册进入批次查看（portrait-viewer mode=batch）时，同一组件需回显收藏态

三、谁可以用收藏？

角色1.0 建议理由
门店成员 ✅ 有 拍摄后在结果页挑片标记；云相册「按收藏」给店员快速找精选批次。
数据写在门店 photos，与现有 Tab 一致。
注册客户 ❌ 1.0 不做 客户分包相册（packageCustomer/pages/gallery）尚无「按收藏」Tab，也无 isFavorite 写入入口。
若开放需单独定义：是「客户私有收藏」还是「门店可见标记」，权限与 UI 不同。
  • 结论（1.0）：收藏 = 门店内部选片工具，仅门店端结果页 / 批次查看页展示收藏按钮。
  • 客户若需「我的喜欢」，建议 2.0 用独立字段如 customerFavorite 或客户维度收藏表，避免与门店选片混淆。
