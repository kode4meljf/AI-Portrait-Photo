设计说明 & 落地要点

  • 状态头 + 四步进度条：与列表状态胶囊一致；当前步高亮，已完成步骤连线着色。
  • 摆台成片区：大图金框；仅图内角标「点击放大」作提示，点击整图 wx.previewImage（不设右上角「全屏预览」，避免重复入口）。
  • 客户卡片：展示 getCustomerDisplayName + 微信昵称/手机号脱敏；可跳转客户详情（若有）。
  • 订单信息：订单号一键复制；展示 createTime / updateTime / storeName。
  • 物流 · 按 status 调序（非 shippingNo）：
    • 待处理 / 制作中：顺序 = 进度条 → 成片 → 客户 → 订单 → 物流空态（置底）。
    • 已发货：状态头下增加物流摘要条（最新一条轨迹）；顺序 = 进度条 → 物流卡片 → 成片 → 客户 → 订单。
    • 已完成：成片优先；物流收成签收摘要置底。
    • 列表「查看物流」进详情时可 scrollIntoView 物流区，无需全局固定物流第一。
  • 底部操作：待处理→确认制作；制作中→查看物流；已发货→确认签收；已完成→仅联系客服/再次下单（可选）。
  • 数据字段：现有 orderNo, status, photoUrl, frameName, styleName, shippingNo, customerInfo, createTime, updateTime；物流轨迹可新增 logisticsCompany, logisticsTraces[] 或云函数聚合查询。
